DogPoint Adopce — security hardening shortlist
==============================================
(From internal review; not a full audit. Update as you ship fixes.)

P0 — Fix before or immediately after production
-------------------------------------------------
[x] /api/upload — `requireAuth` on `POST /api/upload` (upload.ts). Patron/mod flows already send JWT; no role change to avoid breaking staff posts.
[x] /api/email/test — `requireAuth` + `requireAdmin` on `/api/email/*` (emailTest.ts). Admins use Admin login JWT as before.
[x] CORS — Production: if `CORS_*` list empty, fallback to one origin from `FRONTEND_BASE_URL` / `APP_BASE_URL` / `PUBLIC_WEB_BASE_URL`. Dev: still allows any origin when list empty. Set `CORS_ALLOWED_ORIGINS` explicitly for multiple domains (e.g. www + apex).

P1 — High value, do soon
-------------------------
[x] Rate limiting — Implemented in backend: `src/middleware/rateLimiters.ts` + `express-rate-limit`.
    - Login: 30 req / 15 min / IP. Forgot-password: 8 / hour. Reset-password: 20 / hour.
    - Other auth POSTs: 120 / 15 min. Upload POST: 40 / 15 min.
    - Tune via code or add env later; DO/nginx can add a second layer.
[x] XSS — DOMPurify on render: `frontend/src/utils/sanitizeHtml.ts` used for post/notification HTML
    (ModeratorPostsList, ModeratorNewPost preview, NotificationsPage). Optional: also sanitize on API save.
[ ] Secrets — Process only: no real keys in git; use server `.env` / DO secrets; rotate if ever leaked. (Not automatable in repo.)

P2 — Ongoing (reviewed 2026-03-30)
---------------------------------
[ ] npm audit — Run `npm audit` in `backend/` and `frontend/` regularly; test after `npm audit fix`.
    Last check: backend reported many findings (several **high**, often **@aws-sdk/** transitive — fixes available via
    bumping `aws-sdk` / lockfile refresh). Frontend reported **high** on **@remix-run/router** / **react-router-dom** chain
    (advisory: open redirect / XSS class — upgrade path: newer `react-router-dom` when compatible), plus **lodash** / dev
    tooling. Not auto-fixed here (needs regression testing).
[ ] JWT lifetimes — Current code (for awareness): main user login token **7d** (`auth.ts` signToken); **admin moderator**
    token **14d** (`adminModerators.ts`); `utils/jwt.ts` **30d**; several flows **30d** / one **1h**. Shorter TTLs or
    refresh tokens = product change; httpOnly cookies = separate project.
[ ] DB + edge — **Not enforced in git:** DB should not be publicly reachable; app DB user = least privilege; **HTTPS** on
    public host; **HSTS** at CDN/load balancer (DigitalOcean). Confirm in DO dashboard + managed DB network rules.
[ ] API secrets — Stripe/FIO/DO/SMTP/JWT only via deployment env; admin routes use `requireAdmin` / `requireStaff` as
    appropriate (spot-check when adding routes).
