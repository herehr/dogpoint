DogPoint Adopce — security hardening shortlist
==============================================
(From internal review; not a full audit. Update as you ship fixes.)

P0 — Fix before or immediately after production
-------------------------------------------------
[ ] /api/upload — Require auth (and ideally role: staff or ownership). Unauthenticated 200MB uploads to public Spaces = abuse and cost.
[ ] /api/email/test — Require admin auth, or dev-only, or remove in production. Unauthenticated = spam to arbitrary addresses if SMTP is on.
[ ] CORS — Set CORS_ALLOWED_ORIGINS in production. Empty list = all origins allowed (risky with credentials: true).

P1 — High value, do soon
-------------------------
[ ] Rate limiting — Login, register, password reset, and heavy POSTs (e.g. upload). Use express-rate-limit or edge limits.
[ ] XSS — Post/notification HTML rendered with dangerouslySetInnerHTML: sanitize on save or render (e.g. DOMPurify).
[ ] Secrets — No real keys in git; rotate anything ever committed. .env on server only.

P2 — Ongoing
------------
[ ] npm audit (backend + frontend) regularly; patch critical/high.
[ ] Short JWT lifetimes; consider refresh; reduce XSS impact (httpOnly cookies = larger change).
[ ] DB user least privilege; DB not public. HTTPS + HSTS at load balancer.
[ ] Stripe/FIO/DO keys only in environment; review admin routes stay behind requireAdmin / requireStaff.
