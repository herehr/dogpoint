Obdarovaní – Implementační plán

Funkce darování adopce · DogPoint Adopce

Přehled

Umožnit patronům přidat k adopci jednoho nebo více „obdarovaných“ (příjemců daru). Obdarovaný vidí všechny příspěvky adoptovaného zvířete, ale nic dalšího – žádné platby, předplatné ani jiné funkce.

Současná architektura

  • Přístup: GET /api/adoption/me vrací access: { [animalId]: true } pro zvířata, k nimž má uživatel aktivní předplatné.
  • Příspěvky: Načítají se z /api/posts/public; rozhraní je zobrazí jen tehdy, když hasAccess(animalId) je true.
  • Notifikace: notifyUsersAboutNewPost notifikuje uživatele s aktivním předplatným pro dané zvíře.

Co je potřeba udělat

1. Databáze

  • Nový model: SubscriptionGiftRecipient (nebo GiftRecipient):
    • subscriptionId → předplatné
    • email (povinné, pro pozvánku)
    • userId (volitelné, nastaví se při registraci)
    • displayName (volitelné)
    • createdAt
  • Migrace pro přidání tabulky a indexů.

2. Backend

Úkol Popis
Rozšířit getMe Do access zahrnout všechna animalId, kde je uživatel obdarovaným u aktivního předplatného (shoda podle userId nebo email).
Rozšířit notifyNewPost Zahrnout obdarované do seznamu notifikací (v aplikaci + e-mail).
API pro obdarované CRUD operace pro obdarované u předplatného: seznam, přidat (e-mail), odebrat. Pouze majitel předplatného může spravovat.
Propojení při registraci Při registraci uživatele s pozvaným e-mailem propojit userId v SubscriptionGiftRecipient.

3. Frontend

Úkol Popis
Uživatelský dashboard Sekce „Darovat adopci“ / „Obdarovaní“ u každého adoptovaného zvířete: přidat/odebrat obdarované podle e-mailu.
Přístup Bez změn, pokud backend vrací správnou mapu access.
Pozvánkový flow E-mail s pozvánkou a odkazem na registraci (nebo magic link).

4. Okrajové případy

  • Omezení počtu obdarovaných na předplatné (např. 5).
  • Pouze patron může přidávat/odebírat obdarované.
  • Při zrušení předplatného obdarovaní ztratí přístup.
  • Obdarovaný musí mít uživatelský účet (registrace/přihlášení), aby viděl příspěvky.

Fáze Hodiny

Volitelné rozšíření

  • Magic-link pozvánka (bez hesla při první návštěvě): +2–3 h
  • Admin přehled obdarovaných: +1 h
  • Robustnější validace a zpracování chyb: +1 h

Shrnutí

Základ: Nová tabulka, backend přístup + notifikace + CRUD API, frontend UI, pozvánkový e-mail.

Výsledek: Patron přidává obdarované podle e-mailu; obdarovaní se zaregistrují a vidí pouze příspěvky adoptovaného zvířete.

Náročnost: ~8–11 hodin pro základní funkci.
